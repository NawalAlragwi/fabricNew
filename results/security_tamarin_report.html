3. Security Lemmas — Detailed Analysis
🔐 The Tamarin model uses the Dolev-Yao adversary who can intercept all messages, modify contents, replay old messages, forge identities — yet cannot break hash collision resistance or EUF-CMA signature security.

Lemma L1 — exists-trace
Executability
Sanity / Completeness
A trace exists where university issues a certificate, stores it on-chain, and verifier successfully validates it. Confirms the model is not vacuously true.
lemma Executability:
  exists-trace
  "Ex certID #t1 #t2.
    BlockchainStore(certID,'Org1MSP')@#t1
    & VerificationSuccess(certID)@#t2
    & #t1 < #t2"
✅ 3.47s
Lemma L2 — all-traces
Authentication
Access Control
Only Org1MSP (the university) can place records on the blockchain. No other org — including the adversary — can write certificates. Enforced by getCallerMSP() in chaincode.
lemma Authentication:
  "All certID issuer #t.
    BlockchainStore(certID,issuer)@#t
    ==> issuer = 'Org1MSP'"
✅ 2.18s
Lemma L3 — all-traces
StrongAuthentication
RBAC Enforcement
RBAC restriction: every issuance claim must originate from Org1MSP. The restriction is checked at protocol level, not just implementation level, formally proving RBAC correctness.
✅ 4.92s
Lemma L4 — all-traces
Integrity
Data Immutability
Once a certificate hash is written to the ledger, it cannot be changed. UniqueCertificateID restriction prevents overwriting. Prevents man-in-the-middle modification of certificate contents.
lemma Integrity:
  "All certID h1 h2 #t1 #t2.
    LedgerWrite(certID,h1,h1,'Org1MSP')@#t1
    & LedgerWrite(certID,h2,h2,'Org1MSP')@#t2
    ==> h1 = h2"
✅ 1.87s
Lemma L5 — all-traces
PrivateKeySecrecy
Confidentiality
The private signing key sk_U never enters the adversary's knowledge set K(·). Foundation for forgery resistance — keys stored in MSP keystore, never transmitted.
lemma PrivateKeySecrecy:
  "All org sk #t.
    SecretKey(org,sk)@#t
    ==> not(Ex #ta. K(sk)@#ta)"
✅ 6.34s ⭐
Lemma L6 — all-traces (longest)
ForgeryResistance
Primary Security Property
Most critical lemma (6.34s). An attacker attempting to forge a certificate (without sk_U) cannot make it pass blockchain verification. Follows from: key secrecy + EUF-CMA + RBAC.
lemma ForgeryResistance:
  "All fakeID fakeH #ta.
    AttackerAttemptsForgery(fakeID,fakeH)@#ta
    ==> not(Ex #tv.
         VerificationSuccess(fakeID)@#tv)"
✅ 2.76s
Lemma L7 — all-traces
NonRepudiation
Accountability
If a verifier confirms a cert is authentic, a prior blockchain TX must exist. The university cannot deny having issued it. Evidence: immutable TxID stored with every certificate.
✅ 3.21s
Lemma L8 — all-traces
RevocationCorrectness
Revocation Integrity
After revocation at #tr, no verification can succeed at #tv > #tr. The IsRevoked flag is checked before hash comparison in chaincode.
lemma RevocationCorrectness:
  "All certID #tr.
    CertificateRevoked(certID,'Org1MSP')@#tr
    ==> not(Ex #tv.
         VerificationSuccess(certID)@#tv & #tr < #tv)"
✅ 4.56s
Lemma L9 — all-traces
ReplayResistance
Anti-Replay
Attacker cannot replay a certificate for a different student. Each CertID is uniquely bound to student fields — replaying CertID_Alice for Bob requires either a CertID collision or a hash collision.
✅ 1.43s
Lemma L10 — all-traces
HashBinding
Hash Consistency
When VerificationSuccess occurs, presented hash equals stored hash. No verification without exact hash match. This lemma directly validates the hash function binding property for both SHA-256 and BLAKE2b.
lemma HashBinding:
  "All certID h1 h2 #tv.
    HashVerified(certID,h1,h2)@#tv
    ==> h1 = h2"
✅ 2.09s
Lemma L11 — all-traces
IssuerUniqueness
Provenance
A given CertID is always associated with the same issuing org. Combined with authentication lemma L2, every cert has exactly one org of origin — always Org1MSP.
